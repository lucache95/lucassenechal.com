Build the complete email-link-based preference management page where subscribers can update topics, format, delivery time, location, custom sources, SMS opt-in, and unsubscribe -- all without login.

Purpose: Subscribers who click "Update your preferences" in their daily email land on a token-validated page where they can modify all settings. This is the primary self-service tool for subscriber retention and personalization tuning.

Output: Working /preferences page with token validation, data loading, per-section save forms, and unsubscribe danger zone.

  <action>
    Create `lib/schemas/preferences.ts` with four Zod schemas:

    1. `preferencesUpdateSchema` - validates format (enum: 'digest'|'briefing'|'mixed'), deliveryTime (enum: 'morning'|'afternoon'|'evening'), timezone (string), city (string, max 100), smsOptIn (boolean), phone (string, E.164 refine same as onboarding.ts)
    2. `topicsUpdateSchema` - validates subscriberId (uuid), topics (array of strings for subtopic slugs), customTopics (string, max 500)
    3. `sourcesUpdateSchema` - validates subscriberId (uuid), feedUrls (array of url strings with HTTP/HTTPS-only refine, same pattern as onboarding.ts)
    4. `unsubscribeSchema` - validates subscriberId (uuid)

    All schemas include `subscriberId: z.string().uuid()`.

    Create `app/actions/preferences.ts` with `'use server'` directive and four Server Actions:

    1. `updatePreferences(prevState, formData)` - Parse JSON from hidden 'preferencesData' input. Validate with preferencesUpdateSchema. Upsert subscriber_preferences row (format, delivery_time, timezone, city, sms_opt_in, phone). Use Supabase service_role client. Return `{ success?: boolean; error?: string }`.

    2. `updateTopics(prevState, formData)` - Parse JSON from hidden 'topicsData' input. Validate with topicsUpdateSchema. Delete existing subscriber_topics for this subscriber, then insert new rows using slug-to-UUID resolution (same TOPIC_CATEGORIES lookup pattern as app/actions.ts completeOnboarding). Delete existing subscriber_custom_topics, insert new custom topic if non-empty. Return `{ success?: boolean; error?: string }`.

    3. `updateSources(prevState, formData)` - Parse JSON from hidden 'sourcesData' input. Validate with sourcesUpdateSchema. Delete existing subscriber_sources for this subscriber, then insert new rows with feed_url values. Return `{ success?: boolean; error?: string }`.

    4. `unsubscribeAction(prevState, formData)` - Parse subscriberId from hidden 'subscriberId' input. Update subscribers table: set status='unsubscribed', unsubscribed_at=now(). Return `{ success?: boolean; error?: string }`.

    All Server Actions follow existing pattern: createClient with NEXT_PUBLIC_SUPABASE_URL and SUPABASE_SERVICE_ROLE_KEY, try/catch with console.error logging, return error messages on failure.
  </action>
