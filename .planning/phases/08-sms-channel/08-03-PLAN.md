Create `app/api/webhooks/twilio/route.ts` -- a Next.js API route that handles inbound SMS from Twilio.

**Full implementation:**

```typescript
import { NextRequest, NextResponse } from 'next/server'
import twilio from 'twilio'
import { createClient } from '@supabase/supabase-js'
import { generateText } from 'ai'
import { anthropic } from '@ai-sdk/anthropic'
import { smsIntentSchema, SMS_INTENT_SYSTEM_PROMPT, SMS_FALLBACK_RESPONSE } from '@/lib/sms/intent-parser'

function getSupabase() {
  return createClient(
    process.env.NEXT_PUBLIC_SUPABASE_URL!,
    process.env.SUPABASE_SERVICE_ROLE_KEY!
  )
}

function twimlResponse(message?: string): NextResponse {
  const body = message
    ? `<Response><Message>${escapeXml(message)}</Message></Response>`
    : '<Response></Response>'
  return new NextResponse(body, {
    status: 200,
    headers: { 'Content-Type': 'text/xml' },
  })
}

function escapeXml(str: string): string {
  return str
    .replace(/&/g, '&amp;')
    .replace(/</g, '&lt;')
    .replace(/>/g, '&gt;')
    .replace(/"/g, '&quot;')
    .replace(/'/g, '&#39;')
}
```

**POST handler logic:**

1. **Parse form-encoded body:**
```typescript
const body = await request.text()
const params: Record<string, string> = {}
new URLSearchParams(body).forEach((value, key) => { params[key] = value })
```

2. **Validate Twilio signature:**
```typescript
const authToken = process.env.TWILIO_AUTH_TOKEN!
const signature = request.headers.get('x-twilio-signature') || ''
const webhookUrl = `${process.env.NEXT_PUBLIC_SITE_URL}/api/webhooks/twilio`

if (!twilio.validateRequest(authToken, signature, webhookUrl, params)) {
  return new NextResponse('Forbidden', { status: 403 })
}
```

3. **Extract message fields:**
```typescript
const from = params.From       // E.164 phone
const messageBody = params.Body?.trim() || ''
const optOutType = params.OptOutType
const messageSid = params.MessageSid
```

4. **Handle STOP/START/HELP opt-out webhooks (SMS-06):**

For STOP: Look up subscriber by phone number in subscriber_preferences, set `sms_opt_in = false`. Return empty TwiML (Twilio already sent STOP confirmation -- do NOT send another reply per TCPA rules).

For START: Look up subscriber by phone, set `sms_opt_in = true`. Return empty TwiML.

For HELP: Return empty TwiML (Twilio handles HELP response at platform level).

```typescript
if (optOutType === 'STOP' || optOutType === 'START') {
  const supabase = getSupabase()
  await supabase
    .from('subscriber_preferences')
    .update({ sms_opt_in: optOutType === 'START' })
    .eq('phone', from)
  return twimlResponse()
}
```

5. **Look up subscriber by phone number:**
```typescript
const supabase = getSupabase()
const { data: prefs } = await supabase
  .from('subscriber_preferences')
  .select('subscriber_id, sms_opt_in')
  .eq('phone', from)
  .single()

if (!prefs || !prefs.sms_opt_in) {
  return twimlResponse()  // Unknown number or not opted in
}

const subscriberId = prefs.subscriber_id
```

10. **Detect intent via Claude Haiku with AbortController timeout (10s):**

const controller = new AbortController()
const timeout = setTimeout(() => controller.abort(), 10000)

try {
  const { text: intentJson } = await generateText({
    model: anthropic('claude-haiku-4-5'),
    system: SMS_INTENT_SYSTEM_PROMPT,
    messages: [{ role: 'user', content: messageBody }],
    abortSignal: controller.signal,
  })
  clearTimeout(timeout)

  // ... handle intent
} catch {
  clearTimeout(timeout)
  // Fallback on timeout/parse error
}
```

11. **Handle each intent type:**

**question intent (SMS-04):** Call Claude Haiku again with the newsletter content as context + conversation history. System prompt: "You are a helpful assistant for The Daily Briefing newsletter. Answer questions about the subscriber's latest newsletter content. Keep responses under 300 characters for SMS. Be specific and reference the actual content."

**preference_update intent (SMS-05):** Based on `preferenceAction.field`:
- `topics_add`: Insert into subscriber_custom_topics with the value as description. Reply: "Got it! I've added '{value}' to your topics. You'll see it in tomorrow's briefing."
- `topics_remove`: Insert into subscriber_custom_topics with description "REMOVE: {value}" (the research pipeline will interpret this as negative signal). Reply: "Done! I'll stop including '{value}' in your briefings."
- `delivery_time`: Update subscriber_preferences.delivery_time. Map the value to morning/afternoon/evening. Reply: "Updated! Your briefing will now arrive in the {time}."
- `format`: Update subscriber_preferences.format. Map to digest/briefing/mixed. Reply: "Switched to {format} format. You'll see the change tomorrow."

**unknown intent:** Return `SMS_FALLBACK_RESPONSE`.

12. **Store assistant reply in sms_conversations:**

  message_body: reply,
  direction: 'inbound', // logged as inbound context since Twilio sends the reply via TwiML
  status: 'sent',
})
```

14. **Return TwiML with reply:**
```typescript
return twimlResponse(reply)
```
