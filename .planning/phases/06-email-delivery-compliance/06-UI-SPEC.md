# Phase 6 — UI Design Contract

> Visual and interaction contract for Phase 6: Email Delivery & Compliance. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Phase 6 has two UI surfaces:**
> 1. **Email templates** (React Email) -- daily newsletter emails in 3 formats plus fallback, with per-item feedback links and consulting CTAs. This is the subscriber-facing visual deliverable.
> 2. **Admin dashboard** (Next.js page at /admin) -- internal view showing subscriber list, last send status, and error traces (OPS-03). Protected by a simple auth check (not a login system).

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | React Email (`@react-email/components` v1.0.8 already installed) for email templates; native HTML + Tailwind for admin dashboard |
| Icon library | Inline SVG (matching project pattern from Phase 2) |
| Font | System font stack for emails (`-apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, "Helvetica Neue", Arial, sans-serif`); Geist Sans for admin dashboard |

**Rationale:** No shadcn. The project uses Tailwind v4 with CSS custom properties and hand-rolled UI components. Email templates use React Email with inline styles (matching 3 existing templates in `lib/email/`). The admin dashboard uses the same Tailwind utility classes and design tokens as the rest of the site.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, item-to-source-name gap |
| sm | 8px | Compact element spacing, feedback link row padding |
| md | 16px | Default element spacing, item card padding |
| lg | 24px | Section dividers (between newsletter sections), admin table cell padding |
| xl | 32px | Email container horizontal padding, admin card padding |
| 2xl | 48px | Email container vertical padding (top/bottom) |
| 3xl | 64px | Not used in email templates (excessive for email); admin page-level spacing |

**Legacy hardcoded values -- not part of spacing scale:**

The following values are inherited from existing email templates (`welcome-template.tsx`, `plan-delivery.tsx`, `intake-notification.tsx`) and must not be replicated in new spacing decisions. They exist only in the `margin` property of the email container wrapper:

- `margin: "40px auto"` -- vertical centering margin on email container. All 3 existing templates use this value. New Phase 6 email templates must preserve this value for visual consistency with existing emails, but 40px must not be used anywhere else.

Exceptions:
- Email container max-width: 560px (matching existing email templates in `lib/email/`)
- Email container border-radius: 8px (matching existing email templates)

## Typography

### Email Templates (React Email -- inline styles)

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 15px | 400 (regular) | 1.6 | Item summaries, briefing narrative, synthesis text |
| Heading | 24px | 600 (semibold) | 1.3 | Email header (e.g., newsletter title), CTA button text |
| Label | 12px | 600 (semibold) | 1.5 | Section headers (uppercase, 0.05em letter-spacing), feedback link text, item headlines in digest/mixed |
| Small | 13px | 400 (regular) | 1.5 | Source attribution, footer text, preference/unsubscribe links |

**2 weights only: 400 (regular) and 600 (semibold).** The existing templates use 700 for headings, but at 24px size alone provides sufficient hierarchy. New Phase 6 templates use 600 for all bold/emphasis roles. This consolidation also aligns heading weight with the label weight, reducing visual fragmentation.

Source: Adapted from existing email templates (`welcome-template.tsx`, `plan-delivery.tsx`, `intake-notification.tsx`). Existing templates use 600 as the workhorse weight for all labels, section headers, and inline bold elements.

### Admin Dashboard (Tailwind classes)

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 15px (`text-[15px]`) | 400 (regular) | 1.5 | Table cell content, descriptions |
| Label | 13px (`text-[13px]`) | 600 (semibold) | 1.5 | Column headers, status badges, metadata |
| Heading | 20px (`text-xl`) | 600 (semibold) | 1.2 | Page title, section headings |
| Small | 12px (`text-xs`) | 400 (regular) | 1.5 | Timestamps, error trace details |

| Consulting CTA body | "I build automation that eliminates repetitive admin and follow-ups." |
| Consulting CTA button | "See How I Can Help" |

| Unsubscribe link | "Unsubscribe" |
| Footer brand line | "Lucas Senechal \[middot\] The Daily Briefing" |
| Footer compliance line | "You're receiving this because you subscribed at lucassenechal.com" |
| List-Unsubscribe header | RFC 8058 one-click unsubscribe URL (mailto + https) |

### Engagement-Triggered CTA Escalation (MAIL-10)

| Engagement Level | CTA Treatment |
|-----------------|---------------|
| Default (all subscribers) | Soft CTA: single text line + link after content, before footer. "Need AI systems that handle the busywork? See how I can help." |
| Active (3+ clicks in 7 days) | Medium CTA: highlighted box (#f0f9ff background, 8px border-radius, 20px padding) with heading + body + button. |
| Highly engaged (5+ clicks in 7 days OR 2+ feedback interactions) | Same medium CTA box. No further escalation. Do not become aggressive. |

### Fallback Email Copy (QUAL-04)

| Element | Copy |
|---------|------|
| Heading | "Your briefing is light today" |
| Body | "Today's research didn't turn up enough quality results for your topics. This can happen when news cycles are slow or your interests are very specific." |
| Suggestion heading | "A few things that might help:" |
| Suggestion 1 | "Add broader topic categories to cast a wider net" |
| Suggestion 2 | "Check if your custom topics are too narrow" |
| Suggestion 3 | "We'll keep looking -- tomorrow's briefing will likely be fuller" |
| CTA | "Update Your Preferences" (links to preference management page) |

### Empty State Copy (Admin Dashboard)

| Element | Copy |
|---------|------|
| No subscribers heading | "No subscribers yet" |
| No subscribers body | "Subscribers will appear here after they complete the onboarding flow at /newsletter." |

| Element | Copy |
|---------|------|

### Destructive Actions

| Action | Confirmation Approach |
|--------|----------------------|
| Unsubscribe (email link) | Single-click unsubscribe (RFC 8058 compliant). No confirmation dialog -- immediate processing. The unsubscribe endpoint returns a simple confirmation page: "You've been unsubscribed. You won't receive any more emails from The Daily Briefing." with a re-subscribe link. |
| Pause subscriber (admin) | No confirmation needed -- reversible action with "Resume" button. |

### Feedback Link Specification (FDBK-01)

Each newsletter item includes inline feedback links. The links encode subscriber ID, content item ID, and signal type in query parameters.

| Element | Spec |
|---------|------|
| Format | `More like this / Less like this` |
| "More like this" color | #16a34a (green-600) |
| "Less like this" color | #dc2626 (red-600) |
| Separator | ` / ` (space, slash, space) |
| Font | 13px, weight 400 |
| URL pattern | `https://lucassenechal.com/api/feedback?s={subscriber_id}&i={item_id}&v={more\|less}&t={token}` |
| Placement | Below source attribution for each item (digest/mixed) or below each section (briefing) |
| Click behavior | GET request to API endpoint. Returns a simple HTML page: "Got it -- we'll tune your feed." No redirect back to email. |

### Unsubscribe Specification (MAIL-04)

| Element | Spec |
|---------|------|
| In-email link text | "Unsubscribe" |
| Link placement | Footer area, after "Update your preferences", separated by " \| " |
| Link color | #64748b (muted, not accent -- intentionally understated but visible) |
| URL pattern | `https://lucassenechal.com/api/unsubscribe?s={subscriber_id}&t={token}` |
| List-Unsubscribe header | `<mailto:unsubscribe@lucassenechal.com?subject=unsubscribe>, <https://lucassenechal.com/api/unsubscribe?s={subscriber_id}&t={token}>` |
| List-Unsubscribe-Post header | `List-Unsubscribe=One-Click` |
| Landing page copy | "You've been unsubscribed from The Daily Briefing. You won't receive any more emails. Changed your mind? [Re-subscribe here.](#)" |

## Admin Dashboard Layout Specification (OPS-03)

Route: `/admin` (internal, not linked from public navigation)

### Page Layout

```
[Page: max-width 1024px, centered, padding 32px]
  [Heading: 20px/600] Email Delivery Dashboard

  [Stats row: 4 cards in a grid]
    Total Subscribers | Sent Today | Bounce Rate | Failed Sends

  [Section: Subscriber List]
    [Search input: filter by email]
    [Table: email, format, delivery_time, last_sent, status, errors]
    [Each row expandable to show error trace]

  [Section: Recent Send Log]
    [Table: timestamp, subscriber_email, status, duration_ms, error]
    [Paginated: 50 per page]
```

### Stats Cards

| Card | Value Source | Format |
|------|-------------|--------|
| Total Subscribers | `count(*) from subscribers where status = 'active'` | Number |
| Sent Today | `count(*) from send_log where date = today` | Number |
| Bounce Rate | `bounces / total_sends * 100` for last 7 days | Percentage with 1 decimal |
| Failed Sends | `count(*) from send_log where status = 'failed' and date = today` | Number, red if > 0 |

## Email Deliverability Contract (MAIL-05, MAIL-06)

These are not visual but directly affect email rendering decisions:

| Requirement | Specification |
|-------------|--------------|
| Text-to-image ratio | 60%+ text (MAIL-03). No images in newsletter body. Logo optional in header. |
| HTML weight | Under 100KB total email HTML |
| Max width | 560px (matching existing templates) |
| Font stack | System fonts only (no web fonts in email) |
| Responsive | Single-column layout. No media queries needed at 560px max-width. Content reflows naturally on mobile. |
| SPF/DKIM/DMARC | Already configured per Phase 1 (MAIL-05). Verify records before first send. |
| Domain warm-up | Start at 50 emails/day, increase 50% every 3 days until full volume (MAIL-06) |
| Bounce handling | Auto-pause subscriber after 3 consecutive hard bounces |
| Spam complaint | Auto-unsubscribe on spam complaint feedback loop |

## Content Structure Contract (Phase 5 to Phase 6 Interface)

Phase 6 consumes the structured JSON produced by Phase 5. The following shapes are the rendering contract:

### Digest Input (from `newsletter_content.content_json`)
```
{
  subject: string,
  greeting: string,
  items: Array<{ title, summary, url, sourceName }>,
  signoff: string
}
```

### Briefing Input
```
{
  subject: string,
  intro: string,
  sections: Array<{ heading, body, sourceUrls: string[] }>,
  conclusion: string
}
```

### Mixed Input
```
{
  subject: string,
  synthesis: string,
  items: Array<{ title, oneLiner, url, sourceName }>,
  signoff: string
}
```

### Fallback Input
```
{
  intro: string,
  suggestions: string[],
  topicsSearched: string[]
}
```

Source: Phase 5 UI-SPEC content structure contract and `lib/content/schemas.ts` Zod schemas.
