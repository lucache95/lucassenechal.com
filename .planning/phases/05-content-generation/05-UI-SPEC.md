# Phase 5 — UI Design Contract

> Visual and interaction contract for Phase 5: Content Generation. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Phase 5 is a backend-only phase.** There are no browser-rendered pages, components, or interactive elements. The "UI" is the structured content output consumed by Phase 6 (Email Delivery) for rendering into email templates. This contract governs the copywriting, tone, and structural rules for generated newsletter content.

## Copywriting Contract

This is the primary deliverable of this UI-SPEC. Phase 5 generates all newsletter copy programmatically. The following contracts govern tone, structure, and edge cases.

### Voice Rules (CONT-04)

Lucas's voice is encoded in the system prompt. These are the binding rules:

| Rule | Specification |
|------|--------------|
| Tone | Knowledgeable friend sharing what they found interesting today |
| Perspective | First-person-adjacent: "Here's what caught my eye" not "I have curated for you" |
| Sentence structure | Active voice. Short sentences for emphasis. Longer ones for nuance. |
| Humor | Occasionally wry or dry, never forced or frequent |
| Uncertainty | When something is uncertain, say so honestly |
| Enthusiasm | When genuinely exciting, show it -- not hype |
| Source attribution | Casual: "per TechCrunch" or "(via Reuters)" not "According to a report published by..." |

### Banned Phrases

These phrases must never appear in generated content:

- "In today's rapidly evolving..."
- "Let's dive into..."
- "Stay tuned for more updates!"
- "In this edition of your newsletter..."
- "game-changer"
- "revolutionize"
- "landscape" (when used as metaphor)
- "leverage" (as verb)
- "dive into"
- "moving forward"
- "at the end of the day"

### Format Contracts

#### Digest Format (CONT-01)

| Element | Copy Contract |
|---------|--------------|
| Subject line | Specific and curiosity-driven, not clickbait. Under 60 characters. Reference a specific item or theme. |
| Greeting | One casual sentence. Reference the day or a theme across items. No "Good morning" or "Happy Tuesday." |
| Items (5-8) | Each item: rewritten headline (5-12 words, punchier than original) + 2-3 sentence summary (what happened + why it matters). No filler. |
| Source attribution | Source name after each item (human-readable: "TechCrunch" not "techcrunch.com") |
| Sign-off | One brief line. Warm but not sappy. No "until next time" or "stay tuned." |

#### Briefing Format (CONT-02)

| Element | Copy Contract |
|---------|--------------|
| Subject line | Captures the main theme across all results. Under 60 characters. |
| Intro | 2-3 sentences framing the day's theme or connecting dots. Not preamble -- sets genuine context. |
| Sections (2-4) | Each section: topical heading + 2-4 paragraph narrative synthesis. Weave findings together. Cite sources inline as "(via Source)" or "per Source". Draw connections the reader would not see from headlines alone. |
| Conclusion | 1-2 sentences. Can include a forward-looking note or question to ponder. Not generic. |

#### Mixed Format (CONT-03)

| Element | Copy Contract |
|---------|--------------|
| Subject line | Under 60 characters. |
| Synthesis | 3-5 sentences synthesizing the most important findings. What is the thread connecting today's results? Written in Lucas voice. |
| Items (5-8) | Each item: brief headline (5-10 words) + one sentence essential takeaway + source URL + source name. No filler. |
| Sign-off | Brief closing. Warm, not formulaic. |

### Empty State / Fallback (QUAL-04)

When research yields fewer than 3 results, the system bypasses the LLM and returns pre-written fallback content.

| Element | Copy |
|---------|------|
| Subject line | "Your briefing is light today -- here's why" |
| Intro | "Today's research didn't surface enough high-quality results for your topics. This happens occasionally when news cycles are quiet or your interests are very specific." |
| Suggestion 1 | "Add broader topic categories to catch more signals" |
| Suggestion 2 | "Check if your custom topics are too narrow (e.g., 'Rust async runtime benchmarks' vs 'Rust development')" |
| Suggestion 3 | "We'll keep looking -- tomorrow's briefing will likely be fuller" |
| Tone | Honest, not apologetic. Acknowledges the limitation and provides actionable next steps. |

### Error State

| Element | Copy |
|---------|------|
| Internal error (LLM failure) | No email sent to subscriber. Error logged with run-logger. Subscriber receives next day's newsletter normally. No user-facing error copy needed for v1. |
| Partial results (3-7 items when expecting 8+) | Content generated normally with available results. Schema allows min 3 items. No special messaging needed. |

### Destructive Actions

None. Phase 5 is a write-only pipeline (generates and stores content). No delete, update, or overwrite operations that require user confirmation. The UPSERT on `newsletter_content` (replacing same-day content on regeneration) is an internal operation, not user-initiated.

## Content Structure Contract

Phase 5's output is structured JSON stored in the `newsletter_content.content_json` column. This is the schema contract Phase 6 depends on for email rendering:

### Digest Output Shape
```
{
  subject: string (max 60 chars),
  greeting: string,
  items: Array<{
    title: string (5-12 words),
    summary: string (2-3 sentences),
    url: string (valid URL from research results),
    sourceName: string
  }> (min 3, max 8),
  signoff: string
}
```

### Briefing Output Shape
```
{
  subject: string (max 60 chars),
  intro: string (2-3 sentences),
  sections: Array<{
    heading: string,
    body: string (2-4 paragraphs),
    sourceUrls: string[]
  }> (min 2, max 4),
  conclusion: string (1-2 sentences)
}
```

### Mixed Output Shape
```
{
  subject: string (max 60 chars),
  synthesis: string (3-5 sentences),
  items: Array<{
    title: string (5-10 words),
    oneLiner: string (one sentence),
    url: string (valid URL from research results),
    sourceName: string
  }> (min 3, max 8),
  signoff: string
}
```

### Fallback Output Shape
```
{
  intro: string,
  suggestions: string[] (exactly 3),
  topicsSearched: string[]
}
```
